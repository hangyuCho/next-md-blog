- [1] 그룹 및 사용자들의 정보를 보호
- [2] 방어 메커니즘
  - Password Policy(암호 정책)
	  - 비밀번호 강력 → 보안 철저
	  - 옵션
		  - 최소길이 설정
		  - 특정 유형의 글자 사용 요구
			  - 대문자
			  - 소문자
			  - 특수문자
		  - 비밀번호 변경
			  - 허용
			  - 금지
		  - 비밀번호 만료 기간 지정
			  - 예 90일에 1번 변경
		  - 비밀번호 재사용 제한 적용
  - 다요소 인증 MFA(Multi Factor Authentication)
	  - [n] AWS에서 필수적으로 사용하는 메커니즘
	  - MFA란?
		  - [n] 비밀번호 + 사용자가 가지고 있는 보안장치를 사용
	  - AWS MFA 디바이스 옵션
		  - 가상 MFA 디바이스
			  - 특징
				  - 루트계정 IAM사용자 혹은 다른계정 등등 원하는 수 만큼 계정 및 사용자 등록이 가능함(사용하기 편함)
			  - 종류
				  - Google Authenticator
					  - 1개의 휴대폰만 사용가능
				  - Authy
					  - 여러장치에서 사용가능
		  - U2F(보안키 - 물리적 장치)
			  - 특징
				  - Yubico社의 YubiKey → 서드파티
				  - 하나의 보안키 ↔︎ 여러 루트 계정과 IAM사용자를 지원 가능
				  - 1개의 키로 충분함
		- 하드웨어 Key Fob
			- Gemalto社 → 서드파티
				- 특징 : -
			- SurePassID社 → 서드파티
				- 특징
					- AWS GovCloud(미국정부 클라우드 )접근용 특수키
